# Pflege-Prompts

Diese Prompts helfen dabei, die Dokumentation bei fortlaufender Entwicklung konsistent zu halten.

| Prompt | Zweck |
| --- | --- |
| [update-architecture.md](update-architecture.md) | Architektur-Dokumente unter `docs/architecture/` aktualisieren |
| [update-components.md](update-components.md) | einzelne Komponenten- und deklarative UI-Dokumente unter `docs/components/` aktualisieren |

## Verwendung

Den passenden Prompt in eine neue Codex-/AI-Aufgabe kopieren und zusammen mit der aktuellen Änderung ausführen lassen. Bei Änderungen, die sowohl Architektur als auch UI-Komponenten betreffen, beide Prompts verwenden.
